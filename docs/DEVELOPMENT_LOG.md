## 2026-06-13 — Решение по данным волн

### Ситуация

Антон спросил, можно ли нижний код `waveData` вынести в JSON и использовать без бэкенда на GitHub Pages.

### Ответ

Да, можно.

GitHub Pages отдаёт JSON как статический файл.

### Решение

В будущем можно вынести:

```text
public/data/waves.json
```

И загружать:

```js
fetch('data/waves.json')
```

### Почему не делать это сразу

Это уже изменение логики старта игры, потому что данные будут загружаться асинхронно.

Правильнее делать отдельным патчем после документации.

## 2026-06-13 — Идея отделить данные от логики

### Ситуация

`constants.js` сейчас содержит и настройки, и базы данных.

### Возможное улучшение

Разделить:

```text
js/constants.js          — технические константы
public/data/enemies.json — враги
public/data/towers.json  — башни
public/data/waves.json   — волны
```

### Польза

- легче балансировать;
- проще редактировать волны;
- можно не трогать игровой код при правке данных;
- проще проверять ошибки в базе.

### Риск

JSON не поддерживает комментарии, а текущие волны содержат комментарии вроде `// Волна 5`.

### Решение

При переносе комментарии заменить на поля `note`, `number`, `title`.

## 2026-06-13 — Идея стабильных ID

### Ситуация

Волны используют индексы врагов:

```js
{ e: 0, n: 12, d: 0.2 }
```

### Проблема

Если изменить порядок `enemyData`, волны сломаются логически.

### Идея

Добавить врагам `id`:

```json
{ "id": "apple", "name": "Яблоко" }
```

А в волнах использовать:

```json
{ "enemyId": "apple", "count": 12, "delay": 0.2 }
```

### Статус

Идея принята как будущая архитектурная задача, но не выполняется в текущем этапе документации.
