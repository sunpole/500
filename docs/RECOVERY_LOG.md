## 2026-06-13 — Создание документационной базы восстановления

Причина:

Антон попросил максимально наполнить репозиторий документацией, чтобы можно было восстановить проект до любого состояния, понять, что в нём было, что есть сейчас и что сломано.

Что было сломано:

На момент создания этой записи не подтверждено фактическим запуском, что игра сломана. Но проект находился в переходном состоянии после разделения длинного файла на модули и добавления Vite.

Что восстановлено:

Восстановлена память проекта в виде документации:

- идея проекта;
- история из чата;
- сведения о PR;
- Vite;
- структура данных;
- архитектура;
- known issues;
- чек-листы диагностики;
- roadmap;
- changelog;
- текущая карта файлов.

Откуда восстановлено:

- из текущего чата;
- из фактического просмотра файлов репозитория;
- из `index.html`;
- из `js/constants.js`;
- из `js/game.js`;
- из `js/state.js`;
- из `js/gameActions.js`;
- из `js/ui.js`.

Какие файлы изменены:

Добавлены или обновлены документы в `docs/`, `README.md`, `ROADMAP.md`, `CHANGELOG.md`.

Как проверено:

Проверено создание файлов в GitHub. Код игры не запускался в рамках этой записи.

Что осталось проверить:

- `npm install`;
- `npm run dev`;
- `npm run build`;
- запуск игры в браузере;
- ошибки консоли;
- работоспособность кнопок;
- работоспособность волн;
- работоспособность башен.

## Шаблон: восстановление одного файла

Причина:

Файл был сломан после изменения.

Что было сломано:

Указать ошибку, например:

```text
ReferenceError: grid is not defined
```

Что восстановлено:

Например:

```text
Восстановлен js/game.js из резервной ветки.
```

Откуда восстановлено:

```text
backup-before-docs-2026-06-13
```

Какие файлы изменены:

```text
js/game.js
```

Как проверено:

```bash
npm run dev
npm run build
```

Что осталось проверить:

Игровой процесс до 5 волны.

## Шаблон: откат к релизу

Причина:

Проект полностью перестал запускаться.

Что было сломано:

Указать симптомы.

Что восстановлено:

Откат к релизу:

```text
название-релиза
```

Откуда восстановлено:

GitHub Release / tag.

Какие файлы изменены:

Список файлов.

Как проверено:

- проект открывается;
- игра запускается;
- сборка проходит.

Что осталось проверить:

Сравнить изменения после релиза и решить, какие вернуть.

## Правило

Нельзя делать восстановление молча.

После любого восстановления нужно обновить:

- `docs/RECOVERY_LOG.md`;
- `docs/BUGS_AND_FIXES.md`;
- `CHANGELOG.md`;
- при необходимости `docs/CURRENT_STATE.md`.
