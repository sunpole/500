# Снимок памяти разработки проекта 500

Дата: 2026-06-13.

Этот файл создан как дополнительная страховка к `CHAT_CONTEXT.md` и `CHAT_DEEP_ANALYSIS_2026-06-13.md`.

## Назначение

Сохранить краткую, но ёмкую карту того, что нужно помнить при продолжении разработки.

## Что уже известно

- Проект — браузерная tower defense игра.
- Враги — фрукты и овощи.
- Башни — животные.
- Проект был начат как единый HTML + CSS + JS файл.
- После PR знакомого разработчика проект получил модульную структуру и Vite.
- Владелец проекта хочет развивать игру аккуратно, профессионально и с хорошей русской документацией.
- Важна возможность вернуться из нового чата и понять проект без потери контекста.

## Технологическая память

- Текущий стек: HTML, CSS, JavaScript modules, Vite, npm, GitHub.
- Главный файл с данными: `js/constants.js`.
- Важные массивы: `enemyData`, `towerData`, `waveData`.
- Возможный будущий перенос данных: `public/data/*.json`.
- JSON можно использовать на GitHub Pages без бэкенда.
- JSON не поддерживает комментарии.

## Правила работы

- Не удалять существующую документацию.
- Не менять резервные ветки.
- Не делать крупный рефакторинг без отдельного плана.
- Не смешивать баланс, архитектуру, UI и исправления багов.
- Каждый баг записывать.
- Каждое решение записывать.
- Каждый патч фиксировать в `CHANGELOG.md`.

## Самая важная мысль

Проект нужно вести так, чтобы его можно было восстановить не только по коду, но и по истории решений.
