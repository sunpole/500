# Patch 002 — Data, CSS and UI Foundation

Дата: 2026-06-14  
Версия: `1.0.2`  
Ветка: `codex/patch-002-data-css-ui-foundation`

## Цель

Закрепить следующий безопасный слой после стабильной версии `1.0.1`: вынести игровые данные и стили из крупных файлов, восстановить понятные hotkeys и убрать скачок карточки выбранной башни при изменении описания.

## Что изменено

- Версия повышена до `1.0.2` в:
  - `package.json`;
  - `package-lock.json`;
  - `version.json`;
  - `index.html`.
- Данные вынесены из `js/constants.js` в JSON:
  - `public/data/enemies.json`;
  - `public/data/towers.json`;
  - `public/data/waves.json`.
- `js/constants.js` теперь хранит размеры поля и загружает JSON через `import.meta.env.BASE_URL`.
- CSS вынесен из `index.html` в `css/main.css`.
- Стили `#tower-info` также перенесены из inline-атрибута HTML в CSS.
- Hotkeys закреплены:
  - `1-9` — быстрый выбор башен 1-9;
  - `0` — выбор 10-й башни;
  - `Esc` — снять выбор;
  - ПКМ — снять выбор;
  - `F8` — dev mode;
  - `F9` — рестарт;
  - `F10` — сброс прогресса.
- Карточка выбранной башни получила стабильную минимальную высоту, чтобы блок улучшения/продажи не прыгал при разной длине описания.
- `README.md` обновлен: теперь там указано фактическое расположение JSON-данных.

## Что специально не менялось

- Баланс башен, врагов и волн не редактировался вручную.
- Порядок `enemyData`, `towerData` и `waveData` сохранен.
- `index12.html` не удалялся.
- Логика апгрейда, продажи, урона, DOT и лазеров не переписывалась.
- Полноценный progress stepper для апгрейдов не добавлялся в этом патче.

## Риски

- JSON загружается асинхронно через `fetch`; поэтому игру нужно проверять через Vite/dev server или собранную версию, а не простым открытием `index.html` как файла.
- Для GitHub Pages критично, что `vite.config.js` сохраняет `base: '/500/'`, а `constants.js` использует `import.meta.env.BASE_URL`.
- Поздние волны всё еще могут быть тяжелыми по количеству врагов; этот патч не занимается балансом.

```bash
npm run build
```

Первая сборка после начального варианта JSON-загрузки упала с ошибкой Vite/esbuild:

```text
Top-level await is not available in the configured target environment
```

После этого загрузка была переделана на явную функцию `loadGameData()`, которая вызывается перед `init()`.

Финальная сборка:

- `npm run build` проходит.
- Vite выводит только существующее предупреждение про CJS Node API.

Браузерная проверка выполнена на `http://127.0.0.1:4173/500/`:

- старт игры;
- загрузку canvas;
- отображение версии `1.0.2`;
- магазин башен;
- hotkeys `1`, `2`, `3`, `0`, `Esc`;
- покупку башни;
- выбор построенной башни;
- улучшение и продажу;
- старт волны;
- что JSON-данные реально загрузились и волна запускается;
- отсутствие `error`/`warn` в консоли игры во время smoke-проверки.
