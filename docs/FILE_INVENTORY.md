# Инвентаризация файлов проекта 500

Дата: 2026-06-13.

Этот файл фиксирует известные файлы проекта и их назначение.

Инвентаризация нужна, чтобы после поломки или отката понимать, какие файлы должны быть в проекте и за что они отвечают.

## Основные файлы корня

### README.md

Главное описание проекта.

Содержит:

- что это за проект;
- как запустить;
- где документация;
- главные правила разработки;
- ближайшие шаги.

### package.json

Файл npm-проекта.

Содержит:

```json
{
  "name": "500",
  "version": "1.0.0",
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  },
  "devDependencies": {
    "vite": "^5.0.0"
  }
}
```

Назначение:

- `npm run dev` — локальный запуск;
- `npm run build` — сборка проекта.

### index.html

Главная HTML-страница игры.

Содержит:

- meta-теги;
- title;
- встроенные CSS-стили;
- canvas;
- UI-панель;
- кнопку запуска волны;
- таймер волны;
- dev-панель;
- блок информации о башне;
- подключение `js/game.js`.

Ключевая строка:

```html
<script type="module" src="js/game.js"></script>
```

## Папка js

### js/constants.js

Назначение:

- технические константы поля;
- база врагов;
- база башен;
- база волн.

Экспортирует:

- `GRID_SIZE`;
- `CELL_SIZE`;
- `CANVAS_WIDTH`;
- `CANVAS_HEIGHT`;
- `enemyData`;
- `towerData`;
- `waveData`.

Риск:

Файл содержит и технические константы, и большие игровые данные. В будущем данные лучше вынести в JSON.

### js/state.js

Назначение:

Централизованное состояние игры.

Содержит:

- деньги;
- здоровье;
- волну;
- флаги победы/поражения;
- выбранную башню;
- режим строительства;
- devMode;
- таймеры волн;
- сетку;
- путь;
- башни;
- врагов;
- пули.

Риск:

При рефакторинге нужно использовать `state.grid`, `state.towers`, `state.money`, а не несуществующие глобальные `grid`, `towers`, `money`.

### js/game.js

Назначение:

Основной игровой модуль.

Содержит:

- инициализацию;
- игровой цикл;
- обновление таймеров;
- спавнеры волн;
- движение врагов;
- стрельбу башен;
- обработку мыши;
- проверку установки башен;
- отрисовку;
- hotkeys;
- победу/поражение;
- экспорт функций в `window` для inline-кнопок.

Ключевая функция:

```js
init()
```

Старт:

```js
window.onload = () => init();
```

Риски:

- возможные неимпортированные функции;
- возможные обращения к несуществующим глобальным переменным;
- console.log в игровом цикле;
- разные алгоритмы поиска пути.

### js/gameActions.js

Назначение:

Игровые действия и фабрики объектов.

Содержит:

- `Tower()`;
- `Enemy()`;
- `Bullet()`;
- `generateEnemyPath()`;
- `placeTower()`;
- `sellTower()`;
- `spawnEnemy()`;
- `applyDotEffect()`;
- `findPath()`.

Важная особенность:

`spawnEnemy()` масштабирует параметры врагов в зависимости от номера волны.

Риск:

Баланс нельзя оценивать только по `enemyData`, потому что при спавне враги усиливаются.

### js/ui.js

Назначение:

UI-функции.

Содержит:

- сортировку башен по цене;
- показ информации о башне;
- генерацию кнопок башен;
- обновление UI;
- выбор башни;
- снятие выбора;
- создание UI-контейнеров.

Риски:

- `conf.bbulletSpeed` вместо `conf.bulletSpeed`;
- `money` вместо `state.money`;
- `placingTowerCell` вместо `state.placingTowerCell`;
- inline `onclick` требует функций в `window`;
- `upgradeTower()` и `sellTower()` в общей панели вызываются без координат.

## Минимальный набор для восстановления игры

Если проект сломан, но нужно восстановить минимальную игру, проверить наличие:

```text
index.html
package.json
js/constants.js
js/state.js
js/game.js
js/gameActions.js
js/ui.js
```

Если отсутствует хотя бы один из этих файлов, игра может не запуститься.

## Что делать при добавлении нового файла

При добавлении нового файла обновить:

- этот файл `docs/FILE_INVENTORY.md`;
- `CHANGELOG.md`;
- при необходимости `docs/ARCHITECTURE.md`;
- при необходимости `README.md`.
