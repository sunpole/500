## 2026-06-13 — JSON не поддерживает комментарии

Статус: известное ограничение

Описание:

В `waveData` внутри JS можно использовать комментарии:

```js
// Волна 5
```

Если переносить волны в `.json`, такие комментарии вызовут ошибку парсинга.

Причина:

Формат JSON не допускает комментарии.

Решение:

При переносе в JSON нужно заменять комментарии на поля:

```json
{
  "number": 5,
  "note": "Волна 5",
  "enemies": []
}
```

Проверка:

JSON-файл должен открываться и парситься через `response.json()` без ошибок.

Связанные файлы:

- `js/constants.js`;
- будущий `public/data/waves.json`;
- `docs/DATA_STRUCTURE.md`.

## 2026-06-13 — Риск неправильного пути к JSON на GitHub Pages

Статус: известный риск

Описание:

Если вынести волны в JSON и указать неправильный путь, локально или на GitHub Pages игра может не загрузить данные.

Опасные варианты:

```js
fetch('/data/waves.json')
fetch('public/data/waves.json')
```

Причина:

На GitHub Pages проект может находиться в подпапке `/500/`, а Vite копирует `public` в корень итоговой сборки.

Решение:

Для файла:

```text
public/data/waves.json
```

использовать:

```js
fetch('data/waves.json')
```

Проверка:

Проверить:

- `npm run dev`;
- `npm run build`;
- публикацию на GitHub Pages;
- консоль браузера.

Связанные файлы:

- `docs/VITE_NOTES.md`;
- `docs/DATA_STRUCTURE.md`.

## 2026-06-13 — Риск старта игры до загрузки JSON

Статус: будущий риск при переносе данных

Описание:

Если `waveData` вынести из JS в JSON, данные будут загружаться асинхронно через `fetch`. Игра может попытаться стартовать до того, как данные загружены.

Причина:

`fetch` работает асинхронно.

Решение:

Старт игры должен происходить после загрузки данных:

```js
async function initGame() {
  const waveData = await loadWaveData();
  startGame(waveData);
}
```

Проверка:

- нет ошибки `waveData is undefined`;
- первая волна запускается;
- враги появляются;
- консоль чистая.

Связанные файлы:

- будущий модуль загрузки данных;
- будущий `public/data/waves.json`.

## 2026-06-13 — Риск числовых индексов врагов в waveData

Статус: известный архитектурный риск

Описание:

Сейчас волны используют числовые индексы врагов:

```js
{ e: 0, n: 12, d: 0.2 }
```

Это означает врага `enemyData[0]`.

Если изменить порядок врагов, волны начнут выпускать не тех врагов.

Причина:

Связь по индексу зависит от порядка массива.

Решение:

В будущем добавить врагам стабильные `id` и использовать их в волнах:

```json
{ "enemyId": "apple", "count": 12, "delay": 0.2 }
```

Проверка:

После миграции волны должны ссылаться на врагов по `id`, а не по индексу.

Связанные файлы:

- `js/constants.js`;
- будущие JSON-файлы;
- `docs/DATA_STRUCTURE.md`.

## 2026-06-13 — Риск слишком крупных волн в поздней игре

Статус: требует будущего тестирования

Описание:

В текущем `waveData` есть поздние волны с очень большим количеством врагов, например значения `n` больше 100 и даже больше 800.

Это может быть задумано как хаос/поздняя сложность, но также может привести к просадкам FPS или зависанию.

Причина:

Большое количество объектов одновременно может перегружать canvas-отрисовку и игровую логику.

Решение:

Нужно отдельно протестировать поздние волны и записать результаты.

Возможные варианты исправления:

- уменьшить количество врагов;
- увеличить интервал появления;
- оптимизировать отрисовку;
- ограничить максимальное количество активных врагов;
- сделать поздние волны осознанным стресс-тестом.

Проверка:

- пройти до поздних волн;
- смотреть FPS;
- смотреть зависания;
- смотреть консоль;
- проверить слабый компьютер/телефон.

Связанные файлы:

- `js/constants.js`;
- `docs/DATA_STRUCTURE.md`.

## 2026-06-13 — Риск нарушения баланса DOT-башнями

Статус: требует будущей балансировки

Описание:

Башни с ядом могут иметь накопительный урон и большое количество стаков. Например, поздняя башня Скорпион имеет сильный DOT и много стаков.

Причина:

Если яд складывается слишком эффективно, DOT-башни могут стать сильнее всех остальных и сломать баланс.

Решение:

Нужно провести отдельную балансировку:

- проверить ранний яд Геккона;
- проверить средний яд Ежа;
- проверить поздний яд Скорпиона;
- сравнить эффективность с обычными и лазерными башнями.

Проверка:

- пройти первые 10 волн только через яд;
- пройти средние волны через яд;
- проверить поздние волны со Скорпионом;
- сравнить стоимость и эффективность.

Связанные файлы:

- `js/constants.js`.
