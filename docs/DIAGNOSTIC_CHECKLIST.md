# Диагностический чек-лист проекта 500

Этот файл нужен для проверки, что именно работает, а что сломано.

Заполнять его нужно при каждом крупном этапе: после PR, после рефакторинга, после переноса данных, после публикации.

## Как пользоваться

1. Запустить проект.
2. Выполнить пункты по порядку.
3. Отметить статус.
4. Все ошибки перенести в `docs/BUGS_AND_FIXES.md`.
5. После исправлений снова пройти чек-лист.

Статусы:

```text
OK — работает
FAIL — сломано
SKIP — не проверялось
RISK — работает, но есть риск
```

## 14. Минимальный список ошибок для записи

Если при проверке найдено что-то из этого, сразу записать в `docs/BUGS_AND_FIXES.md`:

- `ReferenceError`;
- `TypeError`;
- ошибка импорта модуля;
- белый экран;
- игра не стартует;
- не создаются кнопки;
- не появляются враги;
- нельзя поставить башню;
- не работает стрельба;
- не начисляются деньги;
- зависание;
- ошибка сборки Vite;
- ошибка GitHub Pages.

## 15. Итоговый статус

На момент создания этого чек-листа документация восстановлена частично, но полноценный запуск проекта ещё нужно проверить локально.

Текущий статус проекта нельзя считать подтверждённо рабочим, пока не выполнены:

npm install
npm run dev
