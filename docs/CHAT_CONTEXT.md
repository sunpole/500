## Объяснение PR

PR — это Pull Request, то есть предложение изменений в проект.

Через PR можно:

- посмотреть список изменённых файлов;
- увидеть добавленные и удалённые строки;
- обсудить изменения;
- принять изменения в основную ветку;
- не смешивать работу другого человека сразу с основной версией.

Для этого проекта PR использовался как безопасный способ показать пример разделения кода.

## Объяснение Vite

Vite — это инструмент для разработки и сборки фронтенд-проекта.

Он нужен, чтобы:

- запускать проект локально;
- использовать JavaScript-модули;
- автоматически обновлять страницу при изменениях;
- собирать готовую версию проекта;
- удобнее публиковать проект.

Основные команды:

```bash
npm install
npm run dev
npm run build
```

`npm install` ставит зависимости.

`npm run dev` запускает локальный сервер разработки.

`npm run build` создаёт готовую сборку.

## Важная идея: отделить данные от логики

В чате обсуждался файл:

```text
js/constants.js
```

В нём сейчас находятся:

- размеры поля;
- база врагов;
- база башен;
- база волн.

Антон спросил, можно ли нижний код с базой волн сделать JSON-файлом, отдельно хранить и открывать на GitHub Pages без сервера.

Ответ: да, можно.

GitHub Pages — статический хостинг. Он может отдавать `.json` файлы как обычные файлы. Для этого не нужен бэкенд.

## Будущая идея структуры данных

Возможная структура:

```text
public/data/enemies.json
public/data/towers.json
public/data/waves.json
```

Почему именно `public`:

- в Vite папка `public` копируется в сборку;
- файлы из `public` доступны напрямую;
- после `npm run build` они попадут в итоговый сайт.

Пример загрузки:

```js
export async function loadWaveData() {
  const response = await fetch('data/waves.json');

  if (!response.ok) {
    throw new Error(`Ошибка загрузки waveData: ${response.status}`);
  }

  return await response.json();
}
```

## Важное ограничение JSON

JSON не поддерживает комментарии.

Нельзя так:

```json
[
  // Волна 5
  { "enemies": [] }
]
```

Нужно так:

```json
[
  {
    "wave": 5,
    "note": "Первая волна с Авокадо",
    "enemies": []
  }
]
```

## Важные технические наблюдения

1. `enemyData`, `towerData`, `waveData` сейчас экспортируются из `constants.js`.
2. Если `waveData` вынести в JSON, код игры должен дождаться загрузки данных перед стартом.
3. Если данные нужны сразу при импорте, проще временно оставить их в JS.
4. Перенос в JSON лучше делать отдельным патчем, не вместе с изменением баланса.
5. После переноса нужно проверить GitHub Pages, потому что пути к файлам важны.
