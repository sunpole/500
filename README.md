# 500

Браузерная игра в жанре tower defense: игрок строит башни-животных и защищается от волн врагов-фруктов и овощей.

Проект развивается как учебно-практический фронтенд-проект на HTML, CSS и JavaScript с использованием Vite.

## Кратко о проекте

Изначально проект был длинным единым файлом HTML + CSS + JS. После просмотра знакомым разработчиком был подготовлен Pull Request, где проект разделён на файлы и модули, а также добавлен Vite.

Главная задача сейчас — развивать проект аккуратно, не теряя идеи из чатов, фиксируя баги, решения, планы и изменения в документации.

## Установка и запуск

Перед первым запуском нужно установить зависимости:

```bash
npm install
```

Запуск локальной разработки:

```bash
npm run dev
```

Сборка готовой версии:

```bash
npm run build
```

После сборки готовый сайт появляется в папке:

```text
dist/
```

## Где лежат основные данные

Сейчас основные данные находятся в:

```text
js/constants.js
```

Там хранятся:

- размеры игрового поля;
- `enemyData` — враги;
- `towerData` — башни;
- `waveData` — волны.

В будущем планируется рассмотреть вынос данных в JSON:

```text
public/data/enemies.json
public/data/towers.json
public/data/waves.json
```

Это возможно без бэкенда, потому что GitHub Pages может отдавать JSON как обычные статические файлы.

## Главные правила разработки

1. Не ломать рабочий проект.
2. Перед крупными изменениями делать резервную ветку или релиз.
3. Резервную ветку не трогать.
4. Каждый патч описывать в `CHANGELOG.md`.
5. Каждый баг записывать в `docs/BUGS_AND_FIXES.md`.
6. Идеи и решения фиксировать в `docs/DEVELOPMENT_LOG.md`.
7. Крупные изменения делать маленькими шагами.
8. Не смешивать баланс, архитектуру и визуальные изменения в одном патче.
