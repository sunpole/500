## 2026-06-13 — Зафиксирована идея выноса waveData в JSON

Тип: `docs`

### Что зафиксировано

Обсуждена возможность вынести `waveData` из `js/constants.js` в отдельный JSON-файл.

Решение: это возможно без бэкенда, потому что GitHub Pages может отдавать JSON как статический файл.

Будущий путь:

```text
public/data/waves.json
```

Будущая загрузка:

```js
fetch('data/waves.json')
```

### Важные ограничения

- JSON не поддерживает комментарии.
- Игра должна ждать загрузки данных перед стартом.
- Пути нужно проверять на GitHub Pages.

### Файлы

- `docs/DATA_STRUCTURE.md`;
- `docs/VITE_NOTES.md`;
- `docs/BUGS_AND_FIXES.md`;
- `docs/ARCHITECTURE.md`;
- `ROADMAP.md`.

## Правила будущих записей

Каждое изменение добавлять сверху, под новой датой.

Пример:

```text
## YYYY-MM-DD — Короткое название

Тип: feat/fix/docs/refactor/balance/chore

### Что сделано
...

### Зачем
...

### Проверка
...
```

## Типы изменений

- `docs` — документация;
- `feat` — новая функция;
- `fix` — исправление ошибки;
- `refactor` — изменение структуры без изменения поведения;
- `balance` — правки баланса;
- `chore` — служебные изменения;
- `test` — тесты и проверки.

## Важное правило

Если изменение связано с багом, нужно также обновить:

```text
docs/BUGS_AND_FIXES.md
```

Если изменение связано с планом развития, нужно также обновить:

```text
ROADMAP.md
```

Если изменение меняет архитектуру, нужно также обновить:

```text
docs/ARCHITECTURE.md
```
